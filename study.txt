최초로 commit 하기 전까지는 git은 해당 파일을 추적하지 않음
버전은 그 프로젝트 안의 모든 변경사항을 포함함.
파일 하나의 변경사항만 하나의 버전으로 만들고싶으면 해당 파일만 커밋할 것

** working copy 에 나타난 파일들을 
add 하면 파일들이 index, staging area 로 올라감 
이것들을 commit 하면 저장소에 그 파일들이 하나의 버전으로서 저장이 됨
수정한 파일을 commit 하기전에 수정 전으로 되돌릴수있음 - discard

이미 commit 한 버전을 취소하려면 ?
reset current branch ->히스토리까지 전부 사라짐(hard)
		mixed 를 쓰면 히스토리는 사라져도 수정된 파일 내용은 uncommited 된 채 유지됨
리셋의 세 가지 옵션(hard soft mixed)
revert(reverse commit) - 선택한 버전을 취소하여 그 버전 이전상태로 돌림
-> 버전을 역순으로, 순차적으로 revert 진행해야 함(안그러면 conflict 발생)


** 
reset - 선택한 버전의 상태로 되돌아가는것(버전을 삭제)
revert -선택한 버전을 취소해서 그 이전상태로 돌리는 것(버전을 유지하면서 우리가 선택한 버전으로 commit을 만들어 줌)

branch 
하나의 프로젝트 안에서
안정적인 작업/실험적인 작업 동시에 진행할 때 둘을 분리.
실험적인 작업이 잘 끝났을 때 기존 작업으로 병합하는 것까지 제공 
기본 branch 는 master (여기에 다른 branch 를 합침)
